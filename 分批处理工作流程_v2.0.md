# 图片知识库分批处理工作流程 v2.0

**更新日期**: 2026年4月3日  
**版本**: v2.0  
**变更**: 统一使用 batch_processor.py 进行分批管理

---

## 📋 新工作流程

### 完整处理流程

```
用户: "处理新图片"
    ↓
Step 1: AI 执行 batch_processor.py init
        创建分批会话，扫描所有待处理图片
    ↓
Step 2: AI 执行 batch_processor.py next
        获取下一批次待处理图片
    ↓
Step 3: AI 识别该批次的图片内容
        使用AI视觉识别图片文字
    ↓
Step 4: AI 整理内容、更新文档
        分类整理，更新或创建Markdown文档
    ↓
Step 5: AI 执行 batch_processor.py complete <batch_id>
        标记该批次已完成
    ↓
Step 6: 循环 Step 2-5 直到所有批次完成
    ↓
Step 7: AI 执行 auto_process.py --post-process
        执行归档、生成Word、同步ima等后续操作
```

## 🔧 命令参考

### batch_processor.py 命令

```bash
# 初始化分批会话（扫描图片、创建批次）
python batch_processor.py init

# 获取下一个待处理的批次
python batch_processor.py next

# 标记批次为已完成
python batch_processor.py complete <batch_id>

# 查看处理进度
python batch_processor.py progress

# 重置失败的批次
python batch_processor.py reset

# 清除处理状态
python batch_processor.py clear
```

### auto_process.py 命令

```bash
# 标准模式（扫描→归档→生成Word→同步ima→更新提示词）
python auto_process.py

# 模拟运行（不实际执行）
python auto_process.py --dry-run

# 仅执行同步
python auto_process.py --sync-only

# 后处理模式（仅执行归档、生成Word、同步等）
python auto_process.py --post-process

# 后处理指定文件夹
python auto_process.py --post-process "[示范,游攻略]"
```

## 📁 状态文件位置

```
.workbuddy/memory/batch_state.json
```

状态文件包含：
- 会话ID
- 所有批次信息（pending/processing/completed/failed）
- 每个批次的图片列表
- 处理时间记录
- 重试次数

## 🔄 典型使用场景

### 场景1：正常处理新图片

```bash
# 1. 初始化分批会话
python batch_processor.py init

# 2. 获取第一批
python batch_processor.py next
# 输出: 下一批次: 示范_001

# 3. AI识别该批次图片...

# 4. 标记完成
python batch_processor.py complete 示范_001

# 5. 重复步骤2-4直到所有批次完成

# 6. 执行后续处理
python auto_process.py --post-process
```

### 场景2：处理中断后恢复

```bash
# 查看当前进度
python batch_processor.py progress
# 输出: 已完成: 2/5 批次

# 继续获取下一批次
python batch_processor.py next

# 继续处理...
```

### 场景3：重新开始（清除状态）

```bash
# 清除之前的处理状态
python batch_processor.py clear

# 重新初始化
python batch_processor.py init
```

## ⚠️ 注意事项

1. **分批处理由 batch_processor.py 负责**
   - auto_process.py 不再包含分批逻辑
   - 仅用于执行归档、生成Word、同步ima等后续操作

2. **AI识别环节**
   - 识别图片内容仍由AI完成
   - batch_processor.py 只管理"哪些图片需要处理"的状态

3. **状态文件**
   - 不要手动修改 `.workbuddy/memory/batch_state.json`
   - 使用 `batch_processor.py clear` 清除状态

4. **图片归档**
   - 图片处理完成后会被移动到 `已处理图片/` 目录
   - 归档操作由 `auto_process.py --post-process` 执行
