# 图片知识库转化工具 v2.0 - 优化说明

## 📋 优化概述

本次工具优化针对大量图片处理场景进行了全面升级，主要解决以下问题：

1. **长时间无响应** → 智能分批 + 实时进度反馈
2. **内存占用过高** → 动态批次大小 + 内存监控 + 主动GC
3. **OCR引擎频率限制** → 自动降级 + 多引擎轮换
4. **重复内容** → 实时重复检测
5. **缺乏监控** → 实时进度条 + HTML报告

---

## 🚀 新增功能

### 1. 智能分批策略

**原方案问题**：固定批次大小（5/10/15），未考虑图片大小差异

**优化方案**：
```
图片大小分类：
├── 小图 (<500KB): 25张/批
├── 中图 (500KB-2MB): 12张/批
├── 大图 (2MB-5MB): 6张/批
└── 超大图 (>5MB): 3张/批
```

**优势**：
- 避免大图批次处理时间过长
- 小图批次充分利用并发能力
- 内存使用更均衡

### 2. 并发OCR处理

**原方案**：单线程顺序处理

**优化方案**：
- 多线程并发（默认3个Worker）
- 单张图片超时控制（60秒）
- 错误隔离（单张失败不影响其他）

### 3. 增强OCR引擎管理

**新增功能**：
- **自动降级**：主引擎频率限制时自动切换备用引擎
- **智能重试**：失败自动重试（最多2次）
- **引擎冷却**：频率限制后自动冷却30秒
- **失败计数**：连续失败3次后暂时禁用引擎

**引擎优先级**：
1. 腾讯云 OCR（首选）
2. 百度 OCR（第一备用）
3. 本地 Tesseract（第二备用）

### 4. 重复内容实时检测

**检测方式**：
- 哈希快速检查（精确重复）
- 相似度计算（3-gram Jaccard相似度，阈值85%）

**处理策略**：
- 重复内容标记，不写入文档
- 记录到进度文件，便于统计

### 5. 实时监控与报告

**控制台实时显示**：
```
[████████████████████░░░░░░░░░░░░░░░░] 55.0% | 55/100 | 剩余: 0:02:30 | ✓50 ✗3 ⚡2
```

**HTML报告包含**：
- 处理统计卡片
- 进度条可视化
- 事件日志
- 性能指标

## 🎯 场景选择指南

### 选项1：🚀 增强版处理器（推荐）

**适用场景**：
| 场景 | 说明 |
|-----|------|
| 图片数量多 | 超过 20 张图片 |

| 追求效率 | 希望尽快完成处理 |
| 首次使用 | 不确定图片质量 |

- ✅ 引擎自动降级（一个OCR限制自动换另一个）
- ✅ 重复内容检测（自动去重）
- ✅ 实时进度显示（知道处理到哪里了）
- ✅ 内存监控（防止程序崩溃）

**示例场景**：
> 你有 80 张小红书截图要处理，大小不一，有的清晰有的模糊。选这个！

### 选项2：📦 原始处理器（兼容模式）

**适用场景**：
| 场景 | 说明 |
|-----|------|
| 图片数量少 | 少于 20 张图片 |
| 图片质量一致 | 都是清晰的截图 |
| 快速处理 | 不想等待启动时间 |
| 习惯旧版 | 已经用熟了原来的方式 |
| 简单任务 | 只是几张图，不需要复杂功能 |

**优势**：
- ✅ 启动更快（没有智能分析步骤）
- ✅ 代码简单（出问题容易排查）
- ✅ 内存占用小（单线程处理）

**劣势**：
- ❌ 固定批次大小（大图处理慢）
- ❌ 单线程处理（效率较低）
- ❌ 引擎切换需人工（遇到频率限制要手动切换）
- ❌ 无实时进度（不知道还要等多久）

**示例场景**：
> 你只有 5 张微信读书截图，都很清晰，想快速处理完。选这个！

## 🐛 故障排除

### 问题1：内存占用过高

**解决方案**：
1. 减小 `MAX_WORKERS`（默认3）
2. 减小批次大小配置
3. 降低 `MEMORY_THRESHOLD` 触发更频繁的GC

### 问题2：OCR引擎频繁切换

**解决方案**：
1. 增加 `ENGINE_COOLDOWN` 时间
2. 检查API密钥是否有效
3. 考虑升级API套餐

### 问题3：重复内容检测不准确

**解决方案**：
1. 调整 `similarity_threshold`（默认0.85）
2. 降低阈值可提高召回率（可能误报）
3. 提高阈值可提高精确率（可能漏报）

### 问题4：并发处理出错

**解决方案**：
1. 将 `MAX_WORKERS` 设为1（单线程模式）
2. 检查OCR引擎是否支持并发
3. 查看日志确定具体错误
