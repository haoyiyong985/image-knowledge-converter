# V8.2 修复方案

> 基于 V8.1 实机测试（11张历史学图片）发现的问题

---

## 问题总览

| # | 严重度 | 问题 | 根因 | 影响范围 |
|---|--------|------|------|---------|
| 1 | 🔴 高 | **大模型命名未生效** | `generate_doc_name` 内无日志，无法确认是否被调用；且 `_check_llm_available` 仅检查 TENCENT_SECRET_ID/KEY，混元Lite 可能需要独立的 `HUNYUAN_*` key | 命名质量差的根因 |
| 2 | 🟡 中 | **噪点过滤不全** | `ui_noise_patterns` 缺少多种实际噪点模式：短乱码行、平台UI按钮、输入框文字 | 钱谦益末尾「打电话/帮我写」、始封君「甘o亚a」 |
| 3 | 🟡 中 | **标题层级混乱** | `_is_heading_line` 用 `current_section` 判断 ### vs ####，但对编号格式 `##` 和语义标题 `###` 未区分层级；`#### 虞山先生` 被错误提升 | 钱谦益 `#### 虞山先生`、钱穆 `## 1.` |
| 4 | 🟢 低 | **多列排版覆盖不全** | `_rebuild_multicolumn` 仅匹配 `人名+空格+年代`，但OCR输出中年代和描述可能不在同一行（`朱允\n1368-1398在位`） | 明朝皇帝列表 |
| 5 | 🟢 低 | **OCR误识别无纠正** | 「钱牧斋」→「钱穆斋」导致人物变成钱穆；缺少同义词/别名纠错 | 任何OCR音近字误识别 |

## 修复清单

### 🔴 Fix-1: 大模型命名确认与修复

**根因分析**：
- `_check_llm_available()` 检查 `TENCENT_SECRET_ID/KEY`（OCR用的key），但混元Lite API 可能可以用同一套key
- **真正的问题**：`generate_doc_name` 和 `_llm_extract_topic` 几乎**没有任何日志输出**，无法确认是否真的被调用
- 即使置信度<0.8触发了大模型调用，日志中也不会显示

**修复内容**：

1. **增加详细日志**（便于调试）
   ```python
   # generate_doc_name 中增加
   logger.info(f"[命名] 正则置信度: {conf:.2f}, 策略: {'大模型' if conf < 0.8 else '正则'}")
   if llm_result:
       logger.info(f"[命名] 大模型返回: title={llm_result.get('title')}, category={llm_result.get('category')}")
   else:
       logger.info(f"[命名] 使用正则提取: {topic}")
   ```

2. **_llm_extract_topic 增加日志**
   ```python
   logger.info(f"[LLM] 调用混元Lite... (key前缀: {secret_id[:6]}...)")
   # 成功时
   logger.info(f"[LLM] 混元Lite返回: {result}")
   # 失败时已有debug日志，改为info
   ```

3. **验证混元Lite可用性**：运行一次测试调用，确认 TENCENT_SECRET_ID/KEY 是否可用于混元Lite API

### 🟡 Fix-2: 噪点过滤补充

**遗漏的噪点模式**（从本次测试中发现的）：

| 实际噪点 | 模式 | 说明 |
|---------|------|------|
| `艮拍题答疑` | 未知App UI残留 | 短中文行含特定UI词 |
| `打电话` | 通话按钮 | |
| `帮我写` | AI助手按钮 | |
| `发消息...` | 聊天输入框 | |
| `WMA:` | 音频播放器标记 | |
| `1K` | 字数统计标记 | |
| `^日` | OCR乱码 | |
| `今天我` | 对话残片 | |
| `较长的视` | 视频链接截断 | |
| `公步` | OCR乱码 | |
| `甘o亚a` | OCR乱码 | |
| `方=些世地家力` | OCR乱码 | |
| `危机四伙80` | OCR乱码（应为「危机四伏」） | |

**修复策略**：

```python
# 新增噪点模式
r'^(艮拍|拍题|打电话|帮我写|发消息|发消息\.\.\.)$',    # App UI按钮
r'^WMA[:：]',                                              # 音频标记
r'^\d+K$',                                                # 字数统计
r'^\^[^\s]{1,3}$',                                        # 脱字符开头的OCR乱码
r'^今天我',                                                # 对话残片
r'^较长的视',                                              # 视频链接截断
# 乱码检测（含3+个非中文非英文非数字字符）
r'^(?:[^\u4e00-\u9fff\w\s，。、；：！？""''（）【】《》]){3,}',  # 乱码行
```

同时增加**短行乱码通用检测**：
```python
# 长度<=6且非中文主体的行（含2+个乱码字符）
if len(line) <= 6 and len(re.findall(r'[^\u4e00-\u9fff\w\s]', line)) >= 2:
    is_noise = True
```

### 🟡 Fix-3: 标题层级规范化

**问题分析**：
- 钱谦益：`#### 虞山先生` → `虞山先生` 不是语义标题关键词，但 `_is_heading_line` 把它当成了标题（因为符合长度<35条件）
- 钱穆：`## 1. 早年生活` → 编号格式（`1.`）会匹配 `##` 层级，但应该是 `###`（因为是二级标题下的子标题）

**修复内容**：

1. **`_is_heading_line` 增加内容检查**：不仅是长度<35 + 含关键词，还要排除一些误判：
   - 排除以「的」「了」「着」等助词结尾的非标题行
   - 排除含有冒号后面全是内容的句子

2. **编号格式层级区分**：
   - `## 1.` → 保持 `##`（一级章节）
   - 如果 `current_section` 非空（已在章节内），编号标题降为 `###`

3. **增加 **更精确的标题关键词匹配**：
   - 完整匹配：`早年经历` `仕途坎坷` `情感生活` → 标题
   - 部分匹配：`虞山先生` 不是标题关键词 → 不是标题

### 🟢 Fix-4: 多列排版增强

**当前正则**：
```python
r'^([^\s]{2,4})\s+(\d{3,4}[-—]\d{3,4}.{0,10})$'
```
仅匹配 `朱允 1368-1398在位` 在同一行的情况。

**问题**：OCR可能将年代输出为：
```
朱允
1368-1398在位
```

**修复**：增加跨行检测：
```python
# 检测下一行是否为年代/描述
if not name_year:
    next_line = lines[i+1] if i+1 < len(lines) else ''
    name_year_cross = re.match(r'^([^\s]{2,4})\s*$', line)  # 仅人名
    year_alone = re.match(r'^(\d{3,4}[-—]\d{3,4}.{0,15})$', next_line)  # 下一行为年代
    if name_year_cross and year_alone:
        name = name_year_cross.group(1)
        info = year_alone.group(1)
        rebuilt.append(f"### {name}（{info}）")
        i += 2
        continue
```

### 🟢 Fix-5: OCR别名/音近字纠正（可选）

**当前方案**：在 `_fix_common_ocr_errors()` 中新增别名映射表：

```python
# 常见历史人物别名/音近字纠正
OCR_ALIAS_MAP = {
    '钱穆斋': '钱牧斋',  # 牧→穆
    '朱厚熄': '朱厚熜',  # 熄→熜（嘉靖帝）
    '朱载屋': '朱载垕',  # 屋→垕（隆庆帝）
}
```

**注意**：此方案为启发式，可能误纠正正确内容。建议：
- 仅对**高频已知错误**建立映射
- 纠正后标注 `[OCR纠正: 钱穆斋→钱牧斋]` 便于验证

## 验证方法

修复后用同样的11张图片重新测试（需先恢复环境），逐一对照：
1. 文档命名是否合理（大模型是否生效）
2. 噪点是否被过滤干净
3. 标题层级是否正确
4. 明朝多列排版是否重建
5. 钱穆斋是否被纠正
