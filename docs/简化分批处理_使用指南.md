# 简化分批处理使用指南 v2.0

## 核心改进

### 对比旧版

| 特性 | 旧版 (auto_batch_processor.py) | 新版 (simple_batch_processor.py) |
|------|-------------------------------|----------------------------------|
| **状态管理** | 复杂的JSON state文件 | 无状态管理，直接扫描文件夹 |
| **批次文件** | 生成batch_xxx_info.json | 不生成，AI直接扫描 |
| **图片归档** | 依赖processed_images列表（常为空） | AI传入列表或自动归档所有 |
| **可靠性** | 容易出错，流程脱节 | 简单直接，不易出错 |
| **调试难度** | 难（多组件依赖） | 简单（单一脚本） |

## 使用流程

### 1. AI处理流程

```
用户: 处理新图片
  ↓
AI: 直接扫描待处理图片文件夹
  ↓
AI: 读取所有图片 → OCR识别 → 分类整理
  ↓
AI: 更新Markdown文档
  ↓
AI: 调用简化处理器执行收尾
```

### 2. 收尾流程

```python
# AI处理完成后，调用简化处理器
python scripts/simple_batch_processor.py --finish
```

收尾流程自动执行：
1. 归档所有待处理图片
2. 生成Word文档
3. 同步到ima（强制模式）
4. 更新启动提示词

## 命令说明

| 命令 | 功能 |
|------|------|
| `python scripts/simple_batch_processor.py` | 显示待处理图片状态 |
| `python scripts/simple_batch_processor.py --status` | 显示详细状态 |
| `python scripts/simple_batch_processor.py --finish` | 执行完整收尾流程 |
| `python scripts/simple_batch_processor.py --archive` | 仅归档图片 |

## AI处理提示词

当用户发送"处理新图片"时，AI按以下流程执行：

### 第一步：扫描图片
```python
# 使用Python代码扫描
import os
from pathlib import Path

pending_dir = Path("待处理图片")
images = list(pending_dir.rglob("*"))
images = [f for f in images if f.suffix.lower() in ['.jpg', '.jpeg', '.png', '.webp']]
print(f"找到 {len(images)} 张图片")
```

### 第二步：分批处理
- 每批最多12张
- 读取图片 → OCR识别 → 提取文字

### 第三步：内容分类
- 根据内容自动分类
- 更新现有文档或创建新文档
- 不跳过任何内容

### 第四步：执行收尾
```
python scripts/simple_batch_processor.py --finish
```

## 文件说明

| `scripts/auto_batch_processor.py` | 旧版（已弃用） |
| `generate_word_docs.py` | Word生成脚本（已修复编码） |

| `update_prompt.py` | 启动提示词更新脚本 |

## 注意事项

1. **不再使用** `auto_batch_processor.py`，改用 `simple_batch_processor.py`
2. **不需要** batch_xxx_info.json 文件
3. **不需要** 维护复杂的state状态
4. **直接扫描** 文件夹获取图片列表
5. **统一归档** 所有待处理图片

## 故障排查

### 图片未归档
- 检查 `simple_batch_processor.py` 是否正常运行
- 检查待处理图片文件夹路径是否正确

### Word生成失败
- 检查 `generate_word_docs.py` 是否有emoji字符
- 确保所有文档都有正确编码

### ima同步失败
- 使用 `--force` 模式强制同步
- 检查网络连接
