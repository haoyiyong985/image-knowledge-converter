## P0：核心功能增强（最高优先级）

### P0-1：OCR 三层回退机制

**目标**: 确保任何情况下都能完成识别，不依赖单一服务

**架构设计**:
```
第1层: 腾讯云 OCR → 第2层: 百度 OCR → 第3层: 本地 Tesseract → 第4层: AI 视觉兜底
```

**改进任务**:

| 任务 | 说明 | 状态 |
|------|------|------|
| 封装 Tesseract 自动安装 | Windows/macOS/Linux 一键安装脚本 | 待开发 |
| 完善 OCR 引擎管理器 | 统一接口，自动回退 | 待优化 |
| 添加识别置信度评分 | 低分自动切换引擎 | 待开发 |
| AI 视觉兜底方案 | 作为最终备选 | 待开发 |

**配置示例** (`config/ocr.yaml`):
```yaml
ocr_engines:
  - name: "tencent"
    enabled: true
    priority: 1
    config:
      secret_id: "${TENCENT_SECRET_ID}"
      secret_key: "${TENCENT_SECRET_KEY}"
      region: "ap-beijing"
  
  - name: "baidu"
    enabled: true
    priority: 2
    config:
      app_id: "${BAIDU_APP_ID}"
      api_key: "${BAIDU_API_KEY}"
      secret_key: "${BAIDU_SECRET_KEY}"
  
  - name: "tesseract"
    enabled: true
    priority: 3
    config:
      lang: "chi_sim+eng"
      psm: 6
  
  - name: "ai_vision"
    enabled: true
    priority: 4
    config:
      model: "gpt-4o"
```

**触发条件**:
- 腾讯云 OCR 失败/额度用完 → 自动切换百度
- 百度 OCR 失败/额度用完 → 自动切换本地 Tesseract
- Tesseract 未安装 → 自动尝试安装
- Tesseract 识别置信度 < 0.6 → AI 视觉兜底

### P0-2：可配置分类系统

**目标**: 用户可自定义分类规则，支持新增、修改、删除分类

**架构设计**:
```
图片内容 → 关键词提取 → 分类匹配 → 置信度评分 → 返回分类结果
```

**配置示例** (`config/categories.yaml`):
```yaml
categories:
  - id: "nutrition"
    name: "营养科普"
    keywords: ["抗炎", "营养素", "维生素", "脂肪酸", "抗氧化", "蛋白质"]
    priority: 10
    document: "01_抗炎饮食与营养科普.md"
  
  - id: "gut_health"
    name: "肠道健康"
    keywords: ["益生菌", "膳食纤维", "消化", "排便", "便秘", "肠道"]
    priority: 8
    document: "02_肠道健康与饮食分类.md"
  
  - id: "tcm"
    name: "中医养生"
    keywords: ["食疗", "节气", "汤方", "经络", "穴位", "中药", "体质"]
    priority: 9
    document: "03_中医养生与食疗.md"
  
  - id: "travel"
    name: "旅游攻略"
    keywords: ["旅游", "自驾", "景点", "路线", "攻略", "美食"]
    priority: 7
    document_prefix: "旅游"
  
  - id: "parenting"
    name: "育儿知识"
    keywords: ["宝宝", "育儿", "辅食", "早教", "儿歌", "孕妇"]
    priority: 8
    document: "47_宝宝辅食与育儿.md"

fallback:
  default_category: "misc"
  default_document: "50_综合知识.md"
```

**用户自定义**:
```bash
# 添加新分类
python classify.py add --name "健身运动" --keywords "健身,运动,跑步,力量" --document "健身运动.md"

# 修改分类
python classify.py update --id fitness --keywords "健身,运动,瑜伽,拉伸"

# 删除分类
python classify.py delete --id fitness
```

### P0-3：模板系统

**目标**: 用户可自定义文档输出格式，支持多种模板

**模板目录结构**:
```
assets/templates/
├── default/                    # 默认模板
│   ├── config.yaml           # 模板配置
│   └── style.css             # Word 样式（可选）
├── minimalist/                # 极简模板
│   └── config.yaml
├── detailed/                  # 详细模板
│   └── config.yaml
└── user_custom/              # 用户自定义（不纳入版本控制）
    └── my_template/
        └── config.yaml
```

**模板配置** (`config.yaml`):
```yaml
template:
  name: "默认模板"
  version: "1.0"
  author: "knowledge-converter"

format:
  title:
    prefix: ""                 # 标题前缀，如 "【营养科普】"
    level: 1                   # 主标题层级
    style: "underline"         # 样式: plain/italic/bold/underline
  
  headings:
    levels: [1, 2, 3, 4]      # 支持的标题层级
    auto_numbering: true       # 自动编号
    style: "bold"             # 样式
  
  table:
    style: "grid"             # 样式: grid/zebra/clean
    header_bg: "#f0f0f0"      # 表头背景色
    border: true              # 显示边框
  
  font:
    family: "微软雅黑"
    size: 12
    line_spacing: 1.5

content:
  include_toc: true           # 目录
  include_source: true       # 来源标注
  include_date: true          # 处理日期
  include_page_break: false   # 章节分页

metadata:
  header: |
    ---
    title: "{title}"
    date: {date}
    source: "{source}"
    tags: [{tags}]
    ---
  
  footer: |
    ---
    *数据来源: {source}*
    *处理时间: {datetime}*
    """

output:
  markdown:
    extension: ".md"
    encoding: "utf-8"
  
  word:
    extension: ".docx"
    template: "template.docx"
    page_size: "A4"
```

**使用模板**:
```bash
# 列出可用模板
python generate.py --list-templates

# 使用指定模板
python generate.py --template minimalist

# 创建自定义模板
python generate.py --create-template my_template
```

### P0-4：智能分类器引擎

**目标**: 提高分类准确率，支持模糊匹配和多重分类

**功能特性**:
1. **关键词匹配**: 提取内容关键词，与配置匹配
2. **语义相似度**: 使用 embedding 计算语义相似度
3. **多重分类**: 一张图片可属于多个分类
4. **置信度评分**: 返回置信度，高于阈值自动分类，低则人工确认

**API 设计**:
```python
class ClassifierEngine:
    def classify(self, content: str, threshold: float = 0.7) -> List[ClassificationResult]:

        - document: 目标文档

        - matched_keywords: 匹配的关键词
        """
        
    def suggest(self, content: str, top_k: int = 3) -> List[str]:
        """返回 Top-K 候选分类"""
        
    def train(self, samples: List[Tuple[str, str]]) -> None:
        """使用样本数据训练，提高准确率"""
```

### P1-4：错误处理规范

```markdown
## 错误码规范

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E001 | OCR 引擎不可用 | 检查网络，尝试切换引擎 |
| E002 | OCR 额度用完 | 明天重试，或切换引擎 |
| E003 | 图片格式不支持 | 转换为 JPG/PNG |
| E004 | 分类失败 | 手动指定分类 |
| E005 | 文档写入失败 | 检查文件权限 |
| E006 | ima 同步失败 | 检查凭证，跳过同步 |
| E007 | 网络连接失败 | 检查网络连接 |
| E008 | 文件权限不足 | 管理员权限运行 |

## 自动恢复机制

- OCR 失败 → 自动切换下一个引擎（最多3次）
- 网络超时 → 重试3次，间隔2秒
- 分类模糊 → 列出Top-3候选，AI选择最合适
- 文档冲突 → 备份原文件，合并内容
```
