## 一、分享平台分析

### 1.1 WorkBuddy 官方支持渠道

根据官方文档，WorkBuddy skills 有 **5 种分享方式**：

| 方式 | 适用场景 | 优点 | 缺点 |
|-----|---------|------|------|
| **导出.zip文件** | 小范围分享（朋友、群聊） | 简单直接，无需审核 | 不支持版本管理，不适合规模化 |
| **企业通讯工具** | 团队内部使用 | 即时触达，适合企业 | 仅限协作工具内用户 |
| **Git开源仓库** | 开源社区、技术分享 | 自动化部署，版本可控 | 需要技术背景，受众有限 |
| **腾讯云技能市场** | 大众化推广、商业化 | 官方背书，一键安装，支持付费 | 需要审核，可能收费 |
| **本地目录同步** | 个人多设备管理 | 快捷方便 | 无法分享给他人 |

| 平台 | 类型 | 适用性 | 备注 |

| **GitHub** | 开源社区 | ⭐⭐⭐⭐⭐ | 免费托管，易于发现，适合技术用户 |

### 3.2 SKILL.md 核心内容

```yaml
---
name: image-knowledge-converter
description: Transform mobile screenshots into organized knowledge documents. Extract text from screenshots (Xiaohongshu, WeChat Read, etc.), classify by topic, and merge into Word/Markdown files. Supports batch processing, automatic archiving, and optional ima synchronization. Use this skill when users ask to convert screenshots to documents, organize knowledge base, or build structured knowledge from images.
version: 1.0
author: [Your Name]
tags: [image-processing, knowledge-management, document-generation, ocr, work-automation]
---

## Purpose

This skill automates the conversion of mobile screenshots into structured knowledge documents. It handles text extraction, topic classification, document merging, and image archiving in a fully automated workflow.

## When to Use

Trigger this skill when users request any of the following:
- "Convert screenshots to documents"
- "Organize knowledge base from images"
- "Extract text from screenshots and save as Word"
- "Batch process screenshots"
- "Build knowledge documents from mobile screenshots"
- Merge screenshot content into existing documents
- Related keywords in Chinese: 截图转文档、图片知识库、整理截图、提取图片文字

## Workflow

1. Scan pending images in `待处理图片/<theme>/` directories
2. Extract text from each image using OCR
3. Classify content and determine target document
4. Read existing `.md` file to understand current chapters
5. Append new content without duplicating existing sections
6. Generate updated `.docx` and `.md` files
7. Move processed images to `已处理图片/<theme>/`
8. Run `auto_process.py` to refresh startup prompt
9. Optional: Sync documents to ima (if configured)

## Usage Commands

### Main Commands
- "处理新图片" → Scan and process all pending images
- "处理待处理图片/<theme>" → Process specific theme folder
- "查看现有文档" → List all documents and chapters
- "工具优化" → Improve tool capabilities

### Setup Commands
- "初始化知识库" → Create directory structure and templates
- "配置ima同步" → Set up ima API credentials

## Key Features

- Automatic image scanning and text extraction
- Smart topic classification and document merging
- Duplicate detection to avoid content repetition
- Automatic image archiving
- Dual output: Word + Markdown
- Optional ima synchronization
- Startup prompt auto-refresh
- Extensible template system

## Bundled Resources

### Scripts
- `scripts/auto_process.py` - Scan directories and refresh prompt file
- `scripts/packager.py` - Package skill for distribution
- `scripts/sync_tool.py` - Core processing logic (to be implemented by AI)

### References
- `references/usage-guide.md` - Detailed step-by-step tutorial
- `references/faq.md` - Common issues and solutions
- `references/examples.md` - Use cases and sample workflows

### Assets
- `assets/templates/word_template.docx` - Default Word formatting
- `assets/templates/md_template.md` - Markdown structure
- `assets/prompts/classify_prompt.txt` - Classification AI prompt
- `assets/prompts/extract_prompt.txt` - Content extraction AI prompt

## Dependencies

- Python 3.8+
- python-docx library (pip install python-docx)
- WorkBuddy with image recognition capability
- Optional: ima PC client and API credentials for synchronization

## Troubleshooting

Common issues and solutions are documented in `references/faq.md`.
```

## 四、实施方案

### 阶段一：Skill 创建（1-2天）

1. **初始化 Skill 目录结构**
   ```bash
   python ~/.workbuddy/scripts/init_skill.py image-knowledge-converter
   ```

2. **编写 SKILL.md**
   - 按照 skill-creator 指南编写
   - 包含触发词、工作流、使用说明

3. **迁移核心文件**
   - 将 `auto_process.py` 移入 `scripts/`
   - 创建 `references/` 文档
   - 准备 `assets/` 模板

4. **打包 Skill**
   ```bash
   python ~/.workbuddy/scripts/package_skill.py image-knowledge-converter
   ```

### 阶段二：开源发布（3-5天）

1. **创建 GitHub 仓库**
   - 仓库名：`workbuddy-image-knowledge-converter`
   - 添加 LICENSE（MIT）
   - 编写 README.md

2. **提交代码**
   ```bash
   git init
   git add .
   git commit -m "Initial release v1.0"
   git branch -M main
   git remote add origin https://github.com/yourname/workbuddy-image-knowledge-converter.git
   git push -u origin main
   ```

3. **发布 Release**
   - GitHub 创建 v1.0 Release
   - 上传打包好的 skill.zip
   - 编写 Release Notes

### 阶段三：推广运营（持续）
