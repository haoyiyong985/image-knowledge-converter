## 二、关键问题：V4删除了"历史文化"分类

### V2版本的 THEME_KEYWORDS（正确）:
```python
THEME_KEYWORDS = {
    "中医养生": [...],
    "健康饮食": [...],
    "疾病防治": [...],
    "生活方式": [...],
    "营养科普": [...],
    "旅游攻略": [...],
    "历史文化": ["历史", "文化", "典故", "人物", "朝代", "皇帝", "诗词", "文物", "博物馆", "遗址", "传承"]  # ← V2有！
}
```

### V4/V5/V6版本的 THEME_KEYWORDS（缺失）:
```python
THEME_KEYWORDS = {
    "中医养生": [...],
    "健康饮食": [...],
    "疾病防治": [...],
    "生活方式": [...],
    "营养科普": [...],
    "旅游攻略": [...],
    # "历史文化" ← 被删除了！
}
```

### 影响：
- 包含"历史"、"人物"、"皇帝"等词的内容无法匹配"历史文化"
- 被降级到"综合知识"或错误匹配其他分类
- **钱谦益** 因包含"老人"被归类为"老年养生"
- **周文王** 被归类为"综合知识"

## 四、V4/V6新增的图片归档功能分析

### V4添加的 ImageArchiver 类:
```python
class ImageArchiver:
    def archive_image(self, image_path: str, theme: str = None) -> Optional[str]:
        if theme and theme in THEME_KEYWORDS:
            archive_subdir = self.processed_dir / theme
        else:
            source_folder = src.parent.name  # ← 回退到源文件夹名
            if source_folder not in ['待处理图片', 'images', 'source']:
                archive_subdir = self.processed_dir / source_folder
            else:
                archive_subdir = self.processed_dir / "其他"
```

### 问题分析：

**场景1：历史学文件夹中的图片**
```
源位置: 待处理图片/历史学/钱谦益.jpg
theme检测: "综合知识"（因为"历史文化"被删除）
归档逻辑: theme不在THEME_KEYWORDS → 用源文件夹名
归档位置: 已处理图片/历史学/钱谦益.jpg ✅ 正确
```

**场景2：其他文件夹的图片**
```
源位置: 待处理图片/示范/养生图片.jpg
theme检测: "中医养生"（匹配）
归档位置: 已处理图片/中医养生/养生图片.jpg ✅ 正确
```

**关键发现**：归档逻辑使用源文件夹名作为回退方案，在一定程度上缓解了"历史文化"缺失的问题。

## 五、文档存储问题

### 文档存储使用 theme（而非源文件夹名）:
```python
# V6 文档生成
theme_folder = self.output_dir / theme  # 使用theme作为目录
md_file = theme_folder / f"{safe_name}.md"
```

### 问题：
- 文档存储在 `处理结果/{theme}/` 目录
- 如果theme="综合知识"，文档就会存储到综合知识文件夹
- 而图片归档在 `已处理图片/{源文件夹}/`

### 结果：文档和图片不在同一个分类下

| 内容类型 | 图片归档位置 | 文档存储位置 |
|---------|------------|------------|
| 历史人物（钱谦益）| 已处理图片/历史学/ | 处理结果/综合知识/ |
| 历史人物（周文王）| 已处理图片/历史学/ | 处理结果/综合知识/ |

## 六、同名文档冲突分析

### 问题根源：
1. **V2/V3时代**：文档分散存储，没有合并机制
2. **V5时代**：添加了同名文档检测（基于hash），但：
   - 不同来源的相似内容hash不同
   - 导致同名但内容不同的文档

### 具体案例：

**钱谦益生平**：
```
[综合知识/钱谦益生平.md]
- 来源：Kimi搜索
- 内容：约2000字，详细介绍

[营养科普/钱谦益生平.md]
- 来源：豆包搜索
- 内容：约1000字，简要介绍
```

**原因**：
- 两次处理来自不同的AI工具
- 内容略有差异（详细vs简要）
- hash不同，判定为不同文档
- 但doc_name相同（都叫"钱谦益生平"）

## 七、config/categories.yaml 分析

### 发现：categories.yaml 是独立于 THEME_KEYWORDS 的另一套分类

| 配置 | 用途 | 是否包含"历史" |
|------|------|--------------|
| THEME_KEYWORDS | 决定文档存储目录、图片归档目录 | ❌ 没有"历史文化" |
| categories.yaml | ClassifierEngine 分类参考 | ❌ 没有"历史学" |

### 两个分类体系的冲突：

1. **category**（来自 ClassifierEngine）：
   - 使用 config/categories.yaml
   - 钱谦益 → "老年养生"（因为含"老人"）

2. **theme**（来自 ContentAnalyzer）：
   - 使用 THEME_KEYWORDS
   - 钱谦益 → "综合知识"（因为没有"历史文化"）

3. **文档存储**使用 theme
4. **图片归档**使用 theme（回退到源文件夹）
