## Fix-6: 处理命令零交互（P0）

**现象**：作为 Skill 触发时，每次执行都需要用户手动确认
**根因**：脚本内部的确认逻辑（如果有）+ IDE 层面的 `requires_approval`

**修改内容**：
1. 确认脚本内部无 `input()`/`pause`/`y/n` 等交互代码（已确认没有）
2. SKILL.md 中明确标注「自动化执行，无需确认」
3. 处理命令入口函数 `run_batch_mode` / `run_full_mode` 添加 `--yes`/`-y` 参数跳过确认（如存在确认逻辑）

**改动量**：SKILL.md 更新 + 代码确认（无改动或极少）

## Fix-7: 进度条简化为整体进度（P1）

**现象**：每张图片都输出一行详情 + 进度条，11张图片刷屏
**目标**：只显示整体进度（如 `处理进度: [████████░░] 8/11 (73%) ETA:0m12s`），每张图不再单独打印详情

**修改内容**：
1. `process_single_image` 中删除 `print(f"\n{'─'*60}")` + `print(f"[{index}/{total}] {image_name}")`
2. `progress.log(msg)` 的 `msg` 只在失败时输出，成功时静默
3. `progress.update()` 保留整体进度条
4. 批次处理中也去掉每批次的分隔线 `#####` 头

**改动位置**：
- `process_single_image()` L1924-L1928: 删除分隔线和文件名打印
- `process_single_image()` L2007: 成功时 `progress.log()` → 改为静默更新
- `run_batch_mode()` L2248-L2250: 删除批次分隔线

## Fix-8: 内容整理输出为流畅文档（P1）

**现象**：
- 文档中残留 `###` `##` `####` 等 Markdown 标记（在 Word 和纯文本阅读时不友好）
- 多列排版未完全重建（南宋文档中左右两列内容交错）
- 内容像是 OCR 原文搬运，缺乏连贯性

**根因**：
1. `structure_content()` 给内容添加了 Markdown 标题标记，但 `.md` 文件用 Markdown 查看没问题，**Word 文件**中 `###` 被当作普通文本输出
2. 多列排版重建只处理了「人名+年代」一种模式，未处理通用的左右两列排版

**修改内容**：

### 8a. Word 生成支持 Markdown 标题（`generate_word`）
- `generate_word()` 中解析 `###`/`##` → Word 的 Heading 3/Heading 2 样式
- 目前 `### 标题` 在 Word 里是纯文本 `### 标题`

### 8b. 多列排版通用重建
- 当前只检测 `人名+年代` 模式
- 新增：检测「短行交替模式」（连续短行<20字交替出现 → 可能是多列排版）
- 新增：检测「编号对齐模式」（两列各有序号/符号开头）

### 8c. 去除末尾噪点行（增强）
- 当前过滤规则无法匹配 `甘o亚a`、`方=些世地家力` 等混合乱码
- 新增通用乱码检测：一行中非中文字符（含标点）占比超过50% + 含连续特殊字符 → 删除

**改动位置**：
- `generate_word()` 中添加 Markdown 标题解析逻辑
- `_rebuild_multicolumn()` 新增短行交替检测
- `clean_text()` 新增通用乱码行过滤

## Fix-9: 正则置信度校准（P1）

**现象**：「苏无锡人」置信度 0.90，绕过了 LLM，实际应为「钱穆」
**根因**：`_extract_person_name` 的模式 `(?:江苏|浙江|...)人` 匹配到：
  - 输入行：「苏无锡人」或类似格式
  - 匹配规则：`r'(?:江苏|浙江|...)([^\s，。]{2,4})人'` → 不匹配（因为缺「人」字后缀）
  - 实际匹配到的是：`r'^([^\s，。]{2,4})(?:是|为|...)` 或 `r'(?:江苏)([^\s，。]{2,4})人'` 的变体
  - 结果提取到「苏」或「无锡」作为人名

**修改内容**：
1. **降低人名提取置信度**：`_extract_person_name` 匹配到的人名如果 ≤2 字 → 置信度降为 0.6（触发 LLM）
2. **增加人名验证**：2字人名检查是否是常见姓氏，如果是地名前缀（如「苏」「浙」「鲁」）→ 不做人名
3. **地名前缀黑名单**：`苏/浙/鲁/豫/鄂/湘/赣/闽/粤/桂/黔/滇/蜀/皖/赣` 作为人名时跳过
4. **兜底**：人名提取返回的人名如果和下一行的核心内容不匹配 → 降低置信度

**改动位置**：
- `_extract_person_name()` L509-L515: 添加地名前缀过滤 + 2字人名降权
- `_extract_topic_confidence()` L587-L588: 人名≤2字时返回 0.6 而非 0.9
