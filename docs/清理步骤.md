# 清理 2.0 版本混乱内容步骤

## 问题说明
2.0 版本自动处理时，将 OCR 识别原文直接追加到文档中，导致：
1. 内容格式混乱（代码块、图片文件名等）
2. 分类不准确
3. 与 1.0 版本的 AI 整理内容混在一起

## 清理步骤

### 步骤 1：备份当前文档（可选但建议）
将以下文件复制到备份目录：
- 01_抗炎饮食与营养科普.md
- 02_肠道健康与饮食分类.md
- 03_中医养生与食疗.md
- 04_日常饮食建议.md

### 步骤 2：手动清理或恢复到 3月21日版本
如果之前有备份，可以直接恢复备份文件。

如果没有备份，需要手动删除 2.0 版本添加的内容：

#### 识别 2.0 版本内容的标志：
1. 包含 `## 示范 - 2026-03-18` 或类似标题
2. 包含 `### Camera_xxxx.jpg` 图片文件名
3. 包含 `**识别引擎**: 腾讯云 OCR` 等信息
4. 包含代码块 ``` 包裹的原文

#### 需要删除的内容示例：
```markdown
## 示范 - 2026-03-18 19:07

### Camera_xxxx.jpg

**识别引擎**: 腾讯云 OCR | **置信度**: x.xx

```
OCR识别原文...
```
```

### 步骤 3：删除 2.0 版本生成的新主题文档
删除以下文件（如果有）：
- 06_示范_新主题.md

### 步骤 4：清理进度文件
删除 progress/ 目录下的所有 JSON 文件

### 步骤 5：将图片移回待处理目录
将已处理图片/示范/ 下的图片移回 待处理图片/示范/

## 清理后使用 1.0 版本 workflow

1. 运行扫描脚本：
   ```bash
   python auto_process.py
   ```

2. 在 WorkBuddy 中发送：「处理新图片」

3. AI 会自动：
   - 识别图片内容
   - 整理编辑内容
   - 正确分类
   - 生成规范文档

## 注意事项

- 1.0 版本依赖 WorkBuddy AI 进行内容整理，质量更高
- 2.0 版本的自动分类和整理功能不够完善
- 建议继续使用 1.0 版本 workflow
