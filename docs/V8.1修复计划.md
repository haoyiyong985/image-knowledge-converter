## 二、核心原则：修正处理流程

### 当前V8流程（有问题）
```
图片 → OCR → 轻度噪点过滤 → 正则提取主题(基于OCR原文) → 写文档 → 合并判断
                                      ↑ 命名基于未整理的OCR原文
                         合并时已用了错误的文件名，无法语义合并
```

### 正确流程（V8.1目标）
```
图片 → OCR(原始文本)
        ↓
  ① 内容深度整理（去噪 + 语义标题识别 + 可读化）
        ↓
  ② 基于整理后内容 → AI/大模型提炼主题 + 分类（免费模型）
        ↓
  ③ 合并判断（相似主题合并，用最终主题统一命名）
        ↓
  ④ 生成最终文档（MD + Word）
        ↓
  ⑤ IMA同步 + 图片归档
```

## 六、技术要点

### 内容整理与V2的兼容策略

不是简单复制V2代码，而是**提取V2的核心逻辑移植到V8架构中**：

V2的 `structure_content()` 核心逻辑（第235-273行）：
```python
# V2: 检测标题行 → 分章节组织 → 第一级 ### 标题，后续章节 #### 标题
for line in lines:
    is_heading = self._is_heading_line(line)  # ← 语义判断
    if is_heading and current_section is None:
        structured.append(f"### {line}")        # 第一级标题
    elif is_heading:
        structured.append(f"#### {line}")       # 后续章节标题
```

V8.1合并方案：
```python
def structure_content(self, text):
    for line in lines:
        # 优先用V8的编号格式匹配（1. / 一、/ ● ）
        matched = self._match_number_heading(line)
        if matched: 
            structured_lines.append(matched)
            continue
        # 其次用V2的语义关键词匹配（功效/做法/禁忌/生平/...）
        if self._is_semantic_heading(line):  # ← V2的逻辑
            level = "###" if not has_heading else "####"
            structured_lines.append(f"{level} {line}")
            continue
        # 兜底：普通行
        structured_lines.append(line)
```

### 离线降级保证

整个V8.1在大模型不可用时仍可正常工作：
- 大模型调用失败 → 降级到正则命名
- 内容整理不依赖大模型（纯正则+规则）
- 合并逻辑不依赖大模型（纯字符串相似度）
- 分批处理不依赖任何外部服务
