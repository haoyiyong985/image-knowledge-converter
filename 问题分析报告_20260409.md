## 🚨 严重问题清单

### 问题1：缺少"历史学"分类（最严重）

**问题描述**：
- 分类配置文件 `config/categories.yaml` 中没有"历史学"分类
- 导致所有历史类内容被错误归类

**影响**：
- "钱谦益生平" → 被分类为"老年养生"（因含"老人"关键词）
- "周文王同母弟" → 被分类为"综合知识"
- "明朝历史" → 被分类为"综合知识"
- 历史内容分散到多个不相关的分类

**证据**：
```
分类结果测试：
- "明末清初文学家钱谦益" → 综合知识 (30%)
- "周文王，历史" → 综合知识 (30%)
- "清朝历史，营养知识" → 综合知识 (30%)
- "老年养生，老人" → 老年养生 (23%) ← 错误！
```

### 问题2：同名文档冲突（内容完全不同）

**问题描述**：
- 脚本允许同名文档出现在不同分类文件夹
- 没有验证同名文档的内容是否相同

**影响**：
| 文档名 | 出现位置 | 内容摘要 |
|--------|---------|---------|
| 钱谦益生平 | 综合知识/ | Kimi搜索结果，约2000字 |
| 钱谦益生平 | 营养科普/ | 豆包搜索结果，约1000字 |
| 查一下明末钱穆斋的历史 | 生活方式/ | Kimi搜索结果 |
| 查一下明末钱穆斋的历史 | 育儿教育/ | 豆包搜索结果 |

**根本原因**：
- 文档名由 `ContentAnalyzer.generate_doc_name()` 生成
- 不同AI工具(Kimi/豆包)搜索同一主题，生成不同内容
- 但因为 doc_name 相同（基于标题而非内容），被分散存储

---

### 问题3：分类体系混乱（category vs theme）

**问题描述**：
脚本中存在两套独立的分类体系：

1. **category（分类）**：来自 `ClassifierEngine`
   - 用途：记录原始分类，用于统计
   - 配置：`config/categories.yaml`

2. **theme（主题）**：来自 `ContentAnalyzer.detect_theme()`
   - 用途：决定文档存储路径、图片归档路径
   - 配置：`THEME_KEYWORDS` 字典

**问题**：
- category 和 theme 使用不同的关键词配置
- 导致同一内容被分配不同的分类
- 文档存储在 theme 目录，但统计使用 category

---

### 问题4：归档路径不一致

**当前逻辑**：
```python
def archive_image(self, image_path: str, theme: str = None):
    if theme and theme in THEME_KEYWORDS:
        archive_subdir = self.processed_dir / theme
    else:
        source_folder = src.parent.name  # 使用源文件夹名
        if source_folder not in ['待处理图片', 'images', 'source']:
            archive_subdir = self.processed_dir / source_folder
```

**问题**：
- 图片归档优先使用 theme
- 但如果 theme="综合知识"，则使用源文件夹名
- 导致同一批图片被归档到不同位置

### 问题5：文档命名依赖标题提取

**问题描述**：
```python
def generate_doc_name(self, text: str, category: str, index: int) -> str:
    title = self.extract_title(text)  # 从OCR文本提取标题
    if title:
        return title[:15]
```

**问题**：
- OCR 识别的标题可能不准确
- 不同来源（Kimi vs 豆包）提取的标题不同
- 导致同一内容被命名为不同文档

---

### 问题6：重复检测过于严格

**问题描述**：
- 使用 MD5 hash 检测重复
- 如果同一主题的不同版本内容略有差异，hash 不同
- 但 doc_name 相同，导致生成多个同名文档
