## 🚀 关键优化点详解

### 1. 智能分批策略

```python
# 优化前：固定批次
批次大小 = 5/10/15  # 不考虑图片大小

# 优化后：动态批次
小图 (<500KB)     → 25张/批
中图 (500KB-2MB)  → 12张/批
大图 (2MB-5MB)    → 6张/批
超大图 (>5MB)     → 3张/批
```

**效果**：
- 避免大图批次处理时间过长
- 小图批次充分利用并发能力
- 内存使用更均衡

### 2. 并发处理架构

```
┌─────────────────────────────────────────┐
│           并发处理流程                  │
├─────────────────────────────────────────┤
│  批次图片 → 线程池(3 Workers)           │
│     ↓                                   │
│  并发OCR识别 ← 引擎自动降级             │
│     ↓                                   │
│  内容分类 + 重复检测                    │
│     ↓                                   │
│  结果聚合 → 增量写入文档                │
└─────────────────────────────────────────┘
```

**优势**：
- 单张超时60秒，不影响其他图片
- 错误隔离，单张失败继续处理
- 引擎自动切换，无需人工干预

### 3. OCR引擎管理增强

**自动降级策略**：
```
腾讯云 OCR (首选)
    ↓ 频率限制/失败
百度 OCR (第一备用)
    ↓ 频率限制/失败
本地 Tesseract (第二备用)
    ↓ 失败
等待冷却 → 重试
```

**智能重试机制**：
- 失败自动重试（最多2次）
- 频率限制后冷却30秒
- 连续失败3次后暂时禁用

### 4. 重复内容检测

**双重检测机制**：
1. **哈希检测**：快速检测精确重复
2. **相似度检测**：3-gram Jaccard相似度（阈值85%）

**处理流程**：
```
新内容 → 哈希检查 → 相似度检查 → 是否重复?
            ↓              ↓
         精确重复      相似内容
            ↓              ↓
         跳过处理      标记跳过
```

### 5. 实时监控与报告

**控制台实时显示**：
```
[████████████████████░░░░░░░░░░░░░░░░] 55.0%
处理: 55/100 张
剩余: 0:02:30
成功: 50 | 失败: 3 | 重复: 2
内存: 120MB | 引擎: 腾讯云OCR
```

**HTML报告包含**：
- 📊 统计卡片（总数/成功/失败/重复）
- 📈 进度条可视化
- 📝 事件日志时间线
- ⏱️ 性能指标（耗时/吞吐量）

## 🎯 使用建议

### 场景1：少量图片 (<20张)
**建议**：使用原始处理器或增强版均可
- 差异不明显
- 原始处理器更轻量

### 场景2：中等数量 (20-100张)
**建议**：使用增强版处理器
- 智能分批优势明显
- 并发处理提升效率

### 场景3：大量图片 (>100张)
**建议**：必须使用增强版处理器
- 内存管理防止OOM
- 实时进度监控
- 断点续传支持

### 场景4：超大图片 (>5MB)
**建议**：增强版 + 调整批次配置
- 减小批次大小
- 增加批次间隔
- 监控内存使用

## ⚙️ 配置调优指南

### 内存优化
```python
# 如果内存不足，减小并发数
MAX_WORKERS = 2  # 默认3

# 减小批次大小
BATCH_CONFIG["small"]["batch_size"] = 15  # 默认25

# 降低内存阈值，增加GC频率
MEMORY_THRESHOLD = 70  # 默认80%
```

### 速度优化
```python
# 如果网络稳定，增加并发
MAX_WORKERS = 5  # 默认3

# 减小批次间隔
BATCH_INTERVAL = 0.5  # 默认1秒

# 增加批次大小（小图）
BATCH_CONFIG["small"]["batch_size"] = 30  # 默认25
```

### 稳定性优化
```python
# 增加重试次数
max_retries = 3  # 默认2

# 增加引擎冷却时间
ENGINE_COOLDOWN = 5  # 默认2秒

# 降低失败阈值
max_failures = 2  # 默认3
```

## 📝 已知限制

1. **并发限制**：部分OCR引擎可能不支持高并发，需根据实际情况调整 `MAX_WORKERS`

2. **内存限制**：超大图片（>10MB）仍可能占用较多内存，建议单独处理

3. **网络依赖**：云OCR引擎需要稳定网络连接

4. **重复检测**：相似度检测基于3-gram，对于改写内容可能检测不出
