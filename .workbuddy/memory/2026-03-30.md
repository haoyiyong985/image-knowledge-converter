## 问题排查：再次处理的新图片未自动导入ima（22:03）

### 用户反馈
用户反馈：再次处理的新图片依然没有自动导入ima

### 问题排查过程

#### 1. 检查待处理图片状态
待处理图片为0张（说明图片已处理完成）

#### 2. 直接运行ima同步测试
```bash
python ima_sync_v2.py --force
```
结果：**同步成功！19个文档全部导入ima**

### 发现的问题

**问题不在于ima同步失败，而在于用户看不到同步过程和结果**

`simple_batch_processor.py` 中的 `sync_ima()` 方法使用了 `capture_output=True`，导致：
1. ima同步的输出被捕获但没有显示给用户
2. 用户看不到同步进度和结果
3. 用户误以为没有执行同步

### 修复内容

修复 `simple_batch_processor.py`：
```python
# 修改前: capture_output=True（输出被捕获，用户看不到）
# 修改后: capture_output=False（输出直接显示）
result = subprocess.run(
    ...,
    capture_output=False,  # 改为False，让输出直接显示
    ...
)
```

### 同步结果

| 项目 | 结果 |
|------|------|
| 文档总数 | 19个 |
| 同步成功 | 19个 |
| 新增文档 | 8个（旅游攻略类） |
| 失败 | 0个 |

### 经验总结

1. **问题定位**：用户反馈"未自动导入"时，应先直接测试ima同步功能
2. **输出可见性**：子进程输出被捕获时，用户看不到执行过程，容易产生误解
3. **实时反馈**：重要的流程步骤应该让用户能看到实时输出
