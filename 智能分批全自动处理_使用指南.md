# 智能分批全自动处理 - 使用指南 v1.2

**方案特点**: 固定12张/批次，全自动连续处理，零人工干预

---

## 快速开始

### 用户操作

只需在WorkBuddy中发送：

```

```

```

  ↓

  ↓

```

## 分批规则

| 总图片数 | 批次分配 | 说明 |
|---------|---------|------|
| ≤12张 | 1批 | 直接处理 |
| 13-24张 | 2批 | 12+剩余 |
| 25-36张 | 3批 | 12+12+剩余 |
| ... | ... | 以此类推 |

**核心原则**: 固定12张/批，不区分图片大小，自动连续处理

---

## AI处理步骤详解

### 第1步：扫描和分批

AI收到"处理新图片"指令后：

1. 运行扫描脚本（可选，AI可直接读取文件夹）
2. 统计图片总数
3. 按12张/批计算批次
4. 显示分批计划

### 第2步：批量处理（核心）

对于每一批图片：

```python
# AI自动执行，无需用户确认
for batch in batches:
    # 1. 批量读取图片
    for img in batch:
        read_file(img)
    
    # 2. OCR识别（使用AI视觉或调用OCR工具）
    contents = recognize_all_images(batch)
    
    # 3. 批量分类整理
    classified = classify_contents(contents)
    
    # 4. 批量更新文档
    for doc, content in classified.items():
        update_document(doc, content)
    
    # 5. 显示进度
    print(f"第X批完成，进度: Y/Z")
```

**关键**: 全自动连续处理，不询问"是否继续"

### 第3步：统一收尾

所有批次完成后：

```bash
python scripts/auto_batch_processor.py --finish
```

自动执行：
- 归档所有图片到 `已处理图片/`
- 生成Word文档
- 同步到ima
- 更新启动提示词

## 注意事项

### 1. 超时处理
- 12张/批的设计避免了单次处理过多导致超时
- 如仍超时，可减少为8张/批

### 2. 中断恢复
- 处理中断后可重新运行
- 已处理的图片会自动跳过

### 3. 错误处理
- 单张识别失败 → 记录错误，继续处理
- 整批失败 → 跳过该批，继续下一批
- 分类不确定 → 放入"待分类"文档
